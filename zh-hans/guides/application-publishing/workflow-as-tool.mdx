---
title: 发布为工具
---

> 该功能仅面向 Workflow 类型应用。


开发者可以将构建好的工作流应用发布为可复用的工具，供其它工作流或 Agent 应用调用。

![](https://assets-docs.dify.ai/2025/06/61e28fb5402d6b618f98b4ef74927995.png)

## 核心价值

**模块化复用**：将常用的处理流程封装为工具，避免在多个工作流中重复构建相同的逻辑。比如，你可以将"客户信息验证"工作流应用发布为工具，在不同的业务场景中重复使用。

**团队协作**：不同团队成员可以开发专门的工作流工具，其他人可以直接调用而无需了解内部实现细节。这种分工协作模式能够提升整个团队的开发效率。

**版本管理**：配合应用本身的修改版本管理功能，使得工作流工具同样能够支持版本管理。当业务逻辑需要调整时，只需更新工具本身，所有调用该工具的工作流都会自动获得更新。

**降低复杂度**：复杂的多步骤流程可以封装为简单的工具调用，让主工作流的逻辑更加清晰易懂。

## 发布配置详解

![](https://assets-docs.dify.ai/2025/06/2645307e3eabf4442e7e1f8cf4ac3d04.png)

### 基础信息配置

**工具名称**：使用清晰描述性的名称，让其他开发者能够快速理解工具的用途。建议使用业务领域+功能描述的格式，如"客户数据验证器"、"订单状态同步器"等。

**工具调用名称**：这是系统内部使用的唯一标识符。

**描述信息**：详细说明工具的功能、适用场景和使用注意事项。好的描述应该包括：工具解决什么问题、输入什么数据、输出什么结果、有哪些限制条件等。这些信息对于工具的正确使用至关重要。

![](https://assets-docs.dify.ai/2025/06/6ae50109cef7cf7550aa206210ab3638.png)

## 工作流工具调用方式

### 在工作流中使用

在其它工作流中，添加工具节点，选择 Workflow 类型。将前置节点的输出参数填写至工具的输入参数内，完成嵌入。

### 在 Agent 应用中使用

Agent 应用能够自动发现和使用工作流工具。当 Agent 需要执行特定任务时，它会自动选择合适的工具并传入相应的参数。这种方式特别适合复杂的多步骤任务，Agent 可以根据上下文动态决定使用哪些工具以及如何组合使用。
