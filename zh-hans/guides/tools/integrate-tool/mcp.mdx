## MCP 简介

Model Context Protocol (MCP) 是由 Anthropic 公司开发的开放标准协议，它为 AI 模型与外部工具和数据源之间的交互提供了标准化的接口。

<Info>
  如需详细了解 MCP 协议，请参考 [Model Context Protocol](https://docs.anthropic.com/en/docs/agents-and-tools/mcp)。
</Info>

### Dify 与 MCP 服务器

Dify 平台支持通过 HTTP 协议集成 MCP 服务，使得开发者可以轻松地将各种外部 MCP 服务引入至现有 Dify 应用中。将 MCP 服务器添加至 Dify 后，它将作为 **MCP 工具**被应用内引用。

同时，Dify 应用也同样支持发布为 MCP 服务器供外部调用。详细说明请参考 [工作流 - 发布为 MCP 服务器](/zh-hans/guides/workflow/publish#发布为-mcp-服务器)。

## 添加 MCP 服务至工具

<Steps>
  <Step title="准备 MCP 服务器">
    你需要有一个可访问的 MCP 服务器，它可以是：
    - 自行开发的 MCP 服务
    - 开源社区提供的 MCP 服务
    - 第三方商业 MCP 服务
    
    确保目标 MCP 服务器支持 **HTTP 传输协议**，并已提供可公开访问的互联网链接。
  </Step>
  
  <Step title="配置 MCP 服务器">
    在 Dify 工作空间内，导航至 **工具 → MCP** 页面，点击**添加 MCP 服务器**。
    
    <Frame>
      ![配置 MCP 服务](https://assets-docs.dify.ai/2025/05/abd1fc978ef989ea71b88c69a0fc4c30.png)
    </Frame>
    
    在页面内填写以下信息：
    - **名称和 Icon**：为 MCP 服务指定一个易于识别的名称和标识
    - **服务 URL**：MCP 服务的 HTTP 端点地址
  </Step>
  
  <Step title="授权并连接 MCP 服务器">
    部分 MCP 服务器可能要求授权，按照页面提示完成验证。填写完配置信息后，Dify 将自动尝试连接到 MCP 服务并验证其可用性。成功连接后，系统会显示该 MCP 服务提供的工具和能力列表。
    
    ![](https://assets-docs.dify.ai/2025/06/47e877a60912938c8a26b6f291e11116.png)

    <Warning>
    如果连接验证失败，请检查你提供的 URL 和认证信息是否正确，以及 MCP 服务是否正常运行。
    </Warning>
  </Step>
</Steps>

## 使用 MCP 工具

将 MCP 服务器添加至工具后，你可以在 Dify 平台内的三种应用类型内添加 MCP 工具并调用：

- 在 **Chatflow** / **Workflow** 应用中，添加工具节点并选择 MCP 工具。

> 详细说明请参考[工作流 - 工具](/zh-hans/guides/workflow/node/tools#mcp-工具)。

![](https://assets-docs.dify.ai/2025/05/f96f17ff4b7cf00e83159afd67cb7c7d.png)

- 在 **Agent** 应用中，将 MCP 工具添加到代理的可用工具集。
    
![](https://assets-docs.dify.ai/2025/06/c2bbb6559d1ba442a66340debfb921a9.png)

> 详细说明请参考[构建应用 - Agent](/zh-hans/guides/application-orchestrate/agent#4-集成工具)。

<Info>
  如果你是首次接触 MCP 概念，建议先在 Agent 应用内接入 MCP 工具并查看效果。
</Info>

## 更新 MCP 工具

若 MCP 服务器发生了版本更新，例如新增了工具，点击 MCP 服务器详情页的 Update 进行更新。

![](https://assets-docs.dify.ai/2025/06/f8a1b14d60267172947e3dcf40310382.png)

若更新了 MCP 服务器的在线地址，将自动重新触发授权。

<Warning>
  更新服务器时，有可能会影响到曾添加过该 MCP 工具的应用。
</Warning>
