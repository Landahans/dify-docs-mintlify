工作流应用工具类型指的是将已创建的 Workflow 应用转化为独立工具，便于在其它 Workflow 应用和 Agent 应用类型内直接复用功能逻辑，这一功能特别适合处理复杂的业务逻辑和跨应用场景。

> 仅 Workflow 应用类型支持发布为工具。

该类型工具具备以下优势：

<CardGroup cols={2}>
  <Card title="逻辑复用" icon="recycle">
    避免重复开发相同功能，将经过验证的工作流逻辑在多个应用中复用
  </Card>
  <Card title="简化复杂性" icon="layer-group">
    将多步骤操作封装为单个工具调用，降低应用编排的复杂度
  </Card>
  <Card title="集中管理" icon="gears">
    在一处更新工作流逻辑，所有引用该工具的应用都会自动获得更新
  </Card>
  <Card title="功能共享" icon="share-nodes">
    在团队成员间共享专业功能，促进协作开发
  </Card>
</CardGroup>

## 创建工作流工具

<Steps>
  <Step title="创建工作流">
    在 Dify 平台创建工作流（Workflow）应用类型，确保工作流已经过测试并正常运行。
    
    <Frame>
      ![创建工作流](https://assets-docs.dify.ai/2025/05/d9939ba06ce99f92d1eb37d542027caf.png)
    </Frame>
  </Step>
  
  <Step title="定义输入输出">
    为了让工作流能正确作为工具使用，请确保工作流包含以下信息：
    
    - **输入参数**：明确当前工作流的输入项
    - **输出结果**：该工作流应用能够正确输出数据
  </Step>
  
  <Step title="发布工作流">
    完成工作流设计后，点击右上角的 Workflow as Tool，使应用处于可用状态。只有发布状态的工作流才能作为工具被调用。

    ![](https://assets-docs.dify.ai/2025/06/2645307e3eabf4442e7e1f8cf4ac3d04.png)
  </Step>
</Steps>

## 使用工作流工具

在工作流应用内添加工具节点，选择「Workflow」标签。查看所有可作为工具使用的工作流列表。

![工作流工具列表](https://assets-docs.dify.ai/2025/05/87830f2c2b6744022262655f51491118.png)

在应用的流程中，你现在可以像正常请求工具一样调用该工作流应用。工作流的输入参数和输出结果将作为工具的参数和返回值。
